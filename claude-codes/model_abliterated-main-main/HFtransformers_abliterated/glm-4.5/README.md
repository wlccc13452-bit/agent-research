# GLM-4.5-Air-FP8 消融实验总结

* GLM-4.5-Air-FP8 模型信息：
  * 参数规模 1060 亿
  * 采用 FP8 量化，权重占用存储空间大小 105G
* 硬件环境：
  * GPU：8 * NV GTX 4090 48G
  * 内存：1.8T
  * CPU：AMD EPYC 7542 32-Core Processor

## 消融过程中的一些技术要点

* 与之前小型模型消融实验不同，仅针对某些特定层进行消融无法达到实验目的。
  * 尝试对综合排名前20的层进行消融，直到模型生成内容崩溃也无法达到实验目的。
  * 对除综合排名最末的层（也是模型的第一层）进行消融，无法达到实验目的。
  * 对所有层同时进行消融，才能出现目的现象。
* 由正反提示语激发而计算的 refusal direction 强度较弱，直接利用其进行消融无法达到实验目的。
  * 引入了 `消融强度` 概念，对 refusal direction 进行放大，实际进行消融的值等于 `原始 refusal direction` * `消融强度`。
* 模型推理不能采用之前的 hook 方式。当模型参数规模增大后，需要各层分布式加载。模型推理需要在各GPU之间大量通信，效率降低很多。此时使用 hook 方式会导致性能进一步下降，所以采用了直接对权重进行消融的方式。
* FP8 量化带来的影响
  * 使用 pytorch 对模型进行推理时，会将模型本身 torch.float8_e4m3fn 类型的张量自动转换为 torch.float16，带来以下副作用：
    * 保存模型的时候，模型权重占用的存储空间膨胀到 199G
    * 加载保存后的模型，无法正常进行推理。因为模型张量类型和张量元信息不匹配，导致推理错误。
  * 解决方法
    * 利用原始模型推理计算 refusal direction 后，将 refusal direction 持久化，退出程序，释放掉当前张量被转换的模型。
    * 重新加载原始模型，加载持久化的 refusal direction，执行消融操作。
    * 消融过程中，张量的运算也需要特别注意，torch.float8_e4m3fn 支持的运算符有限。当前的实现是先转换成 float32 进行运算，然后再转换回 float8_e4m3fn
    * 消融后的 float8_e4m3fn 张量取值范围可能会超出合法值，当前的处理是按照最大值与合法边界值的比率进行等比例缩放。另一个可选的方案是对张量进行正则化，但可能导致该层网络输出强度减弱，所以保留了当前的实现方案。

## refusal direction 的生成

当前采用中英文混合提示词，正反各 512 个。利用激活首个 token 的隐藏状态来生成 refusal direction，详细参考代码 `generate_refusal_dir`。

## 消融强度的选择

当前采用人工验证的方式：

* 4.2：有较好的消融效果，对特别强烈的 harmful 提示语会进行拒绝，例如“制造炸弹”等。生成回复质量较高。
* 4.25：有非常好的消融效果，几乎所有 harmful 提示语都会接受。生成回复质量较4.2稍差，出现过循环重复相同内容的情况。

## 文件说明

### 主脚本
- **`ablate_glm.py`**  
  主脚本文件，用于执行模型权重消融操作。支持生成拒绝方向、加载缓存数据、应用消融权重，并保存修改后的模型。可通过命令行参数指定消融强度和操作模式。

### 消融管理
- **`ablation_weight_manager.py`**  
  提供 `RefusalAblationWeightManager` 类，用于管理模型权重的消融操作。实现了多层权重消融的具体逻辑，包括应用拒绝方向和消融强度。

### 数据保存与加载
- **`save_load_refusal_dir.py`**  
  提供保存和加载拒绝方向（refusal direction）及相关元数据的功能。支持将消融数据保存为 `.pt` 文件，并从缓存目录中快速加载最新的消融数据。

### 拒绝方向计算
- **`sort_compute_refusal_dir_hook_mgpu.py`**  
  基于 Hook 机制的拒绝方向计算脚本，适用于多 GPU 环境。捕获模型层的激活状态并生成拒绝方向。

- **`sort_compute_refusal_dir_mgpu.py`**  
  在多 GPU 环境下直接计算拒绝方向，未使用 Hook 机制。适用于需要直接操作模型层激活状态的场景。

- **`sort_compute_refusal_dir_mgpu_quant.py`**  
  支持量化模型（FP8）的拒绝方向计算脚本。解决了 FP8 张量运算的兼容性问题，适用于量化模型的特殊需求。

### 工具函数
- **`utils.py`**  
  提供通用工具函数，例如日志记录、设备管理、数据格式化等，为其他脚本提供辅助功能。

### 消融验证
- **`verify_glm.py`**  
  用于验证模型消融效果。加载提示词数据集，向模型发送请求，解析模型应答，判断是否包含拒绝词汇，并将结果保存到 Excel 文件中。

### 常用命令

```bash
# 生成新的拒绝方向并指定持久化目录：
python ablate_glm.py --generate_refusal_dir --ablation_cache_dir "./custom_cache" 

# 加载最新的拒绝方向并应用消融：
python ablate_glm.py --ablation_strength 4.25
```

## 遗留问题

经验证，模型消融后编码能力有很大的削弱：

1. 编码评测集无法成功执行，会陷入无限循环生成的状态。
2. 与 cline，Claude code 这类比较复杂的编码智能体无法正常配合工作。

要想恢复编码能力，需要针对编码数据集做微调。当前的GPU资源无法胜任此类工作，需要 A100/H100 这类GPU集群才行。
