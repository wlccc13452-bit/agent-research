# 前端CLIENT与测试工具流程对比分析

## 🔍 问题发现

**现象：**
- ✅ WebSocket测试工具很容易读取到行情数据
- ❌ 前端CLIENT加载行情数据可能困难

**原因：** 流程不一致！

## 📊 流程对比

### WebSocket测试工具（旧版）

```
用户输入股票代码
    ↓
连接WebSocket
    ↓
发送订阅请求
    ↓
接收行情推送 ✅
```

**特点：**
- 直接订阅用户输入的股票
- 不依赖持仓数据
- 订阅什么就能收到什么

### 前端CLIENT

```
WebSocket自动连接
    ↓
加载持仓数据 (API)
    ↓
从持仓中提取股票代码
    ↓
自动订阅这些股票
    ↓
接收行情推送
```

**特点：**
- 依赖持仓数据
- 只订阅持仓中的股票
- 如果持仓为空或加载失败，则不会订阅任何股票

## 🚨 关键差异

| 项目 | 测试工具（旧版） | 前端CLIENT |
|-----|----------------|-----------|
| **订阅触发** | 手动输入 | 自动（依赖持仓） |
| **数据来源** | 用户输入 | 持仓文件 |
| **持仓依赖** | ❌ 不依赖 | ✅ 必须依赖 |
| **空持仓情况** | 仍可测试 | 无法订阅 |
| **流程复杂度** | 简单 | 复杂 |

## ⚠️ 前端可能的问题

### 问题1：持仓文件为空或不存在

**影响：** 无法订阅任何股票，收不到行情

**诊断：**
```bash
# 检查持仓文件
cat 自持股票.md

# 或通过API检查
curl http://localhost:8000/api/holdings
```

**解决：** 确保持仓文件有内容

### 问题2：持仓文件格式错误

**影响：** 解析失败，无法提取股票代码

**诊断：** 查看服务器日志

**解决：** 检查持仓文件格式

### 问题3：持仓API调用失败

**影响：** 前端无法获取持仓数据

**诊断：**
```bash
# 测试API
curl http://localhost:8000/api/holdings
```

**解决：** 检查服务器是否正常运行

### 问题4：不在Dashboard页面

**影响：** Dashboard的订阅逻辑不会触发

**解决：** 确保在Dashboard页面

### 问题5：WebSocket连接失败

**影响：** 无法接收任何推送

**诊断：** 查看浏览器控制台Network标签

**解决：** 检查WebSocket连接状态

## 📝 前端代码参考

### Dashboard.tsx 订阅逻辑

```typescript
// 订阅实时行情
useEffect(() => {
  if (isConnected && holdings?.sectors) {
    const codes = holdings.sectors.flatMap((s: any) => 
      s.stocks.map((st: any) => st.code)
    );
    if (codes.length > 0) {
      subscribe(codes);
    }
  }
}, [isConnected, holdings, subscribe]);
```

**关键点：**
- 必须等待 `holdings` 数据加载完成
- 必须等待 WebSocket 连接成功（`isConnected`）
- 只有持仓中有股票才会订阅

### useWebSocket.ts Hook

```typescript
// 自动连接
useEffect(() => {
  connect();
  return () => {
    disconnect();
  };
}, [connect, disconnect]);

// 心跳
pingIntervalRef.current = setInterval(() => {
  if (ws.readyState === WebSocket.OPEN) {
    ws.send(JSON.stringify({ action: 'ping' }));
  }
}, 30000); // 30秒一次心跳
```

**关键点：**
- WebSocket 自动连接
- 每30秒发送心跳
- 断线自动重连
