# 股票数据测试工具 - 完整功能说明

## 📋 功能概览

本测试工具提供两种数据读取方式，满足不同测试需求：

### 🔹 直接读取数据库
- **GUI界面模式**：图形化界面，支持多种数据类型查看
- **命令行测试模式**：快速验证数据库连接和数据完整性

### 🔹 WebSocket连接
- **WebSocket测试模式**：测试实时行情推送功能

## 🛠️ 技术实现

### 直接读取数据库

**代码架构：**
```python
# 导入Server端模块
from database.session import async_session_maker
from services.quote_data_service import QuoteDataService
from database.models import StockRealtimeQuote, StockKLineData, DailyReport

# 使用异步数据库连接
async with async_session_maker() as db:
    # 调用Server端服务
    quote = await quote_service.get_quote_from_db(db, stock_code)
```

**优势：**
- 100%复用Server端代码
- 无需重复实现逻辑
- 保证数据一致性

### WebSocket连接

**代码架构：**
```python
# 连接WebSocket
import websockets
websocket = await websockets.connect("ws://localhost:8000/ws")

# 订阅股票
message = {"action": "subscribe", "stock_code": "600219"}
await websocket.send(json.dumps(message))

# 接收推送
while True:
    data = await websocket.recv()
    message = json.loads(data)
    # 处理消息
```

**消息流程：**
```
客户端                    服务器
  |                         |
  |---- subscribe --------> |
  |                         |
  | <---- subscription -----|
  |                         |
  | <---- quote ------------| (每5秒推送)
  | <---- quote ------------|
  | <---- quote ------------|
```

## ⚠️ 注意事项

### 直接读取数据库
1. ✅ 无需启动Server
2. ✅ 不受交易时间限制
3. ✅ 可查看历史数据
4. ⚠️ 注意数据库锁定（避免同时写入）

### WebSocket连接
1. ❗ 必须启动Backend服务器
2. ⚠️ 只在交易时间推送（9:30-15:00）
3. ⚠️ 推送频率为每5秒
4. ⚠️ 需要额外依赖websockets

## 🔍 故障排除

### 问题1：数据库连接失败
```
[ERROR] 数据库连接失败: no such table
```
**解决：** 检查数据库文件是否存在 `backend/data/stock_peg.db`

### 问题2：WebSocket连接失败
```
[ERROR] 连接失败: [Errno 10061] 无法连接
```
**解决：** 启动Backend服务器 `python backend/main.py`

### 问题3：无数据显示
```
[WARN] 未找到该股票的数据
```
**解决：** 检查数据库中是否有数据

### 问题4：WebSocket不推送
```
监听30秒内未收到消息
```
**解决：**
1. 检查是否在交易时间
2. 检查是否成功订阅
3. 查看服务器日志
