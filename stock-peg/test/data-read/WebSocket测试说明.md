# WebSocket 股票数据测试说明

## 📋 测试模式对比

| 测试模式 | 数据读取方式 | 是否需要Server | 适用场景 |
|---------|------------|--------------|---------|
| GUI界面模式 | 直接读取数据库 | ❌ 否 | 本地数据查看、调试 |
| 命令行测试模式 | 直接读取数据库 | ❌ 否 | 快速验证、自动化测试 |
| WebSocket测试模式 | 通过WebSocket连接 | ✅ 是 | 测试实时推送、网络通信 |

### WebSocket 消息格式

#### 1. 订阅消息（客户端 → 服务器）
```json
{
  "action": "subscribe",
  "stock_code": "600219"
}
```

#### 2. 订阅确认（服务器 → 客户端）
```json
{
  "type": "subscription",
  "action": "subscribed",
  "stock_code": "600219",
  "timestamp": "2026-03-08T19:30:00"
}
```

#### 3. 行情推送（服务器 → 客户端）
```json
{
  "type": "quote",
  "stock_code": "600219",
  "data": {
    "code": "600219",
    "name": "南山铝业",
    "price": 7.44,
    "change": -0.32,
    "change_pct": -4.12,
    "open": 7.58,
    "high": 7.67,
    "low": 7.25,
    "volume": 3128140,
    "amount": 23456789.0,
    "timestamp": "2026-03-08T19:30:05"
  },
  "timestamp": "2026-03-08T19:30:05"
}
```

#### 4. 心跳检测
```json
// 客户端发送
{"action": "ping"}

// 服务器响应
{
  "type": "pong",
  "timestamp": "2026-03-08T19:30:10"
}
```

## 🔧 技术实现

### WebSocket 连接

```python
import websockets
import json

# 连接
websocket = await websockets.connect("ws://localhost:8000/ws")

# 订阅
message = {"action": "subscribe", "stock_code": "600219"}
await websocket.send(json.dumps(message))

# 接收消息
while True:
    data = await websocket.recv()
    message = json.loads(data)
    # 处理消息
```

### 消息类型

- `subscription` - 订阅确认/取消
- `quote` - 行情数据推送
- `pong` - 心跳响应
- `alert` - 预警消息

## ⚠️ 注意事项

### 1. 服务器必须运行

WebSocket 模式需要 Backend 服务器运行：
```
错误示例：
[ERROR] 连接失败: [Errno 10061] 由于目标计算机积极拒绝，无法连接

解决方法：
启动Backend服务器
```

### 2. 交易时间推送

实时行情只在交易时间推送：
- 上午：9:30 - 11:30
- 下午：13:00 - 15:00
- 周一到周五（非节假日）

非交易时间可能不会收到行情推送。

### 3. 推送频率

服务器每5秒推送一次行情数据。

### 4. 连接超时

默认监听超时30秒，可通过参数调整：
```python
await client.listen(timeout=60, max_messages=20)
```

## 🆚 与直接读取数据库的对比

### 直接读取数据库（GUI/命令行模式）

**优点：**
- ✅ 无需启动Server
- ✅ 读取速度快
- ✅ 可查看历史数据
- ✅ 不受交易时间限制

**缺点：**
- ❌ 无法测试实时推送
- ❌ 无法测试WebSocket功能

### WebSocket模式

**优点：**
- ✅ 测试实时数据推送
- ✅ 验证WebSocket功能
- ✅ 测试网络通信
- ✅ 模拟真实客户端

**缺点：**
- ❌ 需要Server运行
- ❌ 受交易时间限制
- ❌ 需要额外依赖（websockets）
