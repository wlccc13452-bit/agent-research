# 数据库表结构修复报告

## 问题描述

测试工具在查询每日报告时出现错误：

```
❌ 查询报告失败: (sqlite3.OperationalError) no such column: daily_reports.smart_analysis
```

## 根本原因

数据库表结构过旧，缺少模型中新增的字段：
- `smart_analysis` - 智能分析原始回复 (JSON)
- `smart_analysis_formatted` - 格式化后的分析
- `pmr_data` - PMR 数据 (JSON)
- `llm_model` - 使用的 LLM 模型
- `llm_provider` - LLM 提供商

这些字段是在 `backend/database/models.py` 中定义的，但数据库表未同步更新。

## 修复过程

### 1. 检查数据库结构

创建并运行 `check_db_schema.py`：

```python
# 检查 daily_reports 表结构
cursor.execute("PRAGMA table_info(daily_reports)")
```

**发现缺失字段：**
```
[MISSING] smart_analysis - 缺失

[MISSING] pmr_data - 缺失
[MISSING] llm_model - 缺失
[MISSING] llm_provider - 缺失
```

### 2. 更新数据库表结构

执行 ALTER TABLE 添加缺失字段：

```sql
ALTER TABLE daily_reports ADD COLUMN smart_analysis TEXT;
ALTER TABLE daily_reports ADD COLUMN smart_analysis_formatted TEXT;
ALTER TABLE daily_reports ADD COLUMN pmr_data TEXT;
ALTER TABLE daily_reports ADD COLUMN llm_model VARCHAR(50);
ALTER TABLE daily_reports ADD COLUMN llm_provider VARCHAR(50);
```

## 技术要点

### 1. 完全复用 Server 端代码

测试工具导入并使用 Server 端的：
- `database.session.async_session_maker` - 数据库会话
- `services.quote_data_service.QuoteDataService` - 行情服务
- `database.models` - 数据模型

### 2. 数据库迁移策略

使用 `ALTER TABLE ADD COLUMN` 实现向后兼容的迁移：
- 不破坏现有数据
- 新字段允许 NULL
- 不影响现有功能

### 3. 编码问题处理

Windows 控制台编码问题解决方案：
- 移除所有 emoji 字符
- 使用纯文本标识 `[OK]`、`[ERROR]`、`[WARN]`
