# 前端"暂无K线数据"问题排查指南

## 问题现象

前端显示"暂无K线数据"，但后端API测试正常。

## 已验证 ✅

1. ✅ 后端API正常工作
2. ✅ 数据格式正确（data + metadata）
3. ✅ 本地数据可用（60条K线数据）
4. ✅ API响应快速（平均25ms）

## 排查步骤

### 步骤1: 检查浏览器控制台

**操作**：
1. 打开前端页面（http://localhost:5173）
2. 按F12打开开发者工具
3. 切换到Console标签

**检查内容**：
- ❌ 是否有红色错误信息？
- ❌ 是否有网络请求失败（Network标签）？
- ❌ 是否有API请求404或500错误？

**常见错误**：
```
❌ Failed to fetch
❌ CORS error
❌ 404 Not Found
❌ Unexpected token < in JSON
```

### 步骤2: 检查网络请求

**操作**：
1. 在开发者工具切换到Network标签
2. 刷新页面
3. 筛选XHR请求
4. 查找以下请求：
   - `/api/holdings/` - 获取持仓列表
   - `/api/stocks/kline-db/{stockCode}` - 获取K线数据

**检查内容**：
- ✅ 请求是否发出？
- ✅ 响应状态码是否200？
- ✅ 响应数据是否包含`data`数组？
- ❌ 是否有请求被取消？

### 步骤3: 检查React Query缓存

**操作**：
1. 在开发者工具Console中运行：
```javascript
// 查看React Query缓存
window.__REACT_QUERY_DEVTOOLS_GLOBAL_QUERY_CLIENT__
```

**解决方案**：
```javascript
// 清除所有缓存
queryClient.clear()

// 或清除特定缓存
queryClient.invalidateQueries(['kline-quick'])
queryClient.invalidateQueries(['kline-full'])
```

### 步骤4: 检查路由参数

**操作**：
检查URL是否包含正确的股票代码

**正确的URL格式**：
```
http://localhost:5173/stock/600219
http://localhost:5173/dashboard (会自动选择第一只股票)
```

**检查代码**：
```typescript
// StockDetail.tsx
const { stockCode } = useParams<{ stockCode: string }>();
console.log('当前股票代码:', stockCode);
```

### 步骤5: 检查Loading状态

**可能的问题**：
前端卡在loading状态，没有进入渲染阶段

**检查代码**：
```typescript
// StockDetail.tsx 第315行
if (isLoading || (loadingState.stage !== 'idle' && loadingState.stage !== 'complete')) {
  return <LoadingProgress ... />;
}
```

**调试方法**：
在Console中运行：
```javascript
// 查看当前loading状态
console.log('Loading Stage:', window.loadingStage);
```

### 步骤6: 强制刷新数据

**操作**：
1. 按`Ctrl + Shift + R`（Windows）或`Cmd + Shift + R`（Mac）硬刷新
2. 或清除浏览器缓存：
   - Chrome: F12 → Application → Clear site data
   - Firefox: F12 → Storage → Clear

### 步骤7: 检查股票代码映射

**可能的问题**：
前端传递的股票代码格式不正确

**检查代码**：
```typescript
// holdingsApi.getHoldings()返回的数据格式
{
  sectors: [{
    name: "铝",
    stocks: [{
      code: "600219",  // 确认这个字段存在
      name: "南山铝业",
      sector: "铝"
    }]
  }]
}
```

### 步骤8: 添加调试日志

**在StockDetail.tsx中添加**：

## 快速解决方案

### 方案1: 重启前端服务

```bash
# 停止前端服务
Ctrl + C

# 清除依赖缓存
cd frontend
rm -rf node_modules/.vite

# 重启服务
npm run dev
```

### 方案2: 清除React Query缓存

**在浏览器Console中运行**：
```javascript
// 找到React Query客户端
const queryClient = window.__REACT_QUERY_DEVTOOLS_GLOBAL_QUERY_CLIENT__;

// 清除所有缓存
queryClient.clear();

// 刷新页面
location.reload();
```

### 方案3: 使用无痕模式

**操作**：
1. 按`Ctrl + Shift + N`（Chrome）或`Ctrl + Shift + P`（Firefox）打开无痕窗口
2. 访问 http://localhost:5173
3. 检查K线是否正常显示

### 方案4: 检查前端配置

**检查frontend/.env**：
```env
VITE_API_BASE_URL=http://localhost:8000
```

**检查frontend/vite.config.ts**：
```typescript
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
      }
    }
  }
})
```
