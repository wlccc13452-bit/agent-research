# 前端"暂无K线数据"问题完整诊断报告

## 问题现象
- 后端API测试正常，返回60条K线数据
- 前端显示"暂无K线数据"或"本地暂无可用数据"
- API响应格式正确：`{data: [], metadata: {...}}`

## 根本原因分析

### 1. React Query缓存问题 ⚠️
**症状**: 前端缓存了旧数据（可能是之前数据库为空时的数据）

**原因**:
```typescript
// StockDetail.tsx 第225行
staleTime: 30000, // 30秒内不重新请求
```
React Query会缓存数据，如果之前请求过空数据，30秒内不会重新请求。

### 2. Loading状态卡住 ⚠️
**症状**: 前端卡在loading状态，无法进入渲染阶段

**代码位置**:
```typescript
// StockDetail.tsx 第315-325行
if (isLoading || (loadingState.stage !== 'idle' && loadingState.stage !== 'complete')) {
  return <LoadingProgress ... />;
}
```

**问题**:
- 如果API请求失败，loading状态可能不会正确更新
- 如果error被设置，但loading状态没有正确切换

### 3. Error处理逻辑 ⚠️
**代码位置**:
```typescript
// StockDetail.tsx 第219-222行
} catch (error) {
  setError('获取K线数据失败');
  throw error; // 抛出错误，React Query进入error状态
}
```

**问题**:
- 错误被抛出，但没有触发loading状态完成
- React Query的error状态没有被正确处理
