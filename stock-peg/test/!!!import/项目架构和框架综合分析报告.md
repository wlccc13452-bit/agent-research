## 1. 项目概览

### 1.1 项目定位

**核心设计理念**: 以用户实际持仓（自持股票.md）为核心数据源，构建个性化的智能股票分析平台。

**关键特性**:
- ✅ **用户驱动**: 以用户真实持仓为唯一数据源，而非全市场扫描
- ✅ **实时监控**: 实时行情推送、K线图表、技术指标
- ✅ **多维分析**: 基本面、技术面、市场情绪、国际联动
- ✅ **AI 辅助**: LLM 智能分析、预测模型
- ✅ **数据本地化**: 数据库优先策略，减少网络依赖

### 3.2 数据获取策略

#### 3.2.1 数据库优先策略（✅ 已实现）

```python
# 原则：优先从本地数据库读取，网络API作为后台更新源
# 1. 前端请求 → 后端从数据库读取 → 立即返回
# 2. 数据缺失 → 触发后台更新任务 → WebSocket推送更新
# 3. 避免阻塞前端，提升响应速度
```

**优势**:
- ⚡ 响应速度快（<100ms）
- 🔄 无网络依赖
- 📊 数据可控性强
- 🚀 不受API限制影响

**实现文件**:
- `backend/services/quote_data_service.py`
- `backend/services/stock_data_service.py`
- `backend/services/background_updater.py`

#### 3.2.2 多级缓存策略

```
请求 → 内存缓存 → 数据库 → 网络API → WebSocket推送
```

**缓存层级**:
1. **内存缓存** (LRU, TTL=60s): 热点数据
2. **数据库缓存** (SQLite): 持久化数据
3. **文件缓存** (Markdown): 用户配置

**实现文件**:
- `backend/services/extended_cache.py` - 多种缓存管理器
- `backend/services/kline_cache.py` - K线缓存
- `backend/services/quote_cache.py` - 行情缓存

        while self._running:
            current_modified = os.path.getmtime(self.file_path)
