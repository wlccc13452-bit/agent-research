# 🛠️ Harness Engineering System

> High-density governance framework for AI agent behavior, project state, and engineering standards.

---

## 📂 System Hierarchy

| Layer | Files | Purpose | Context Weight |
|-------|-------|---------|----------------|
| **Executive** | `AGENTS.md` | Hard prohibitions & core constraints | Highest (every turn) |
| **State** | `progress.md`, `decisions.md` | Current tasks & architectural truths (D0XX) | High |
| **Operational** | `skills/` | Executable workflows via commands | On-Demand |
| **Reference** | `reference/` | Deep technical knowledge & historical logs | Low |

---

## ⚡ Core Commands

| Command | Purpose |
|---------|---------|
| `/harness` | Initialize context: `progress → decisions → core-facts → AGENTS` |
| `/maintenance` | System entropy reduction: clean, validate, prune, manage system logs |
| `/dispute` | Adversarial audit: Builder vs. Auditor debate, auto-generate audit report |
| `/update-memory` | Sync memory: update progress, decisions, create session summary |

### Command Details

#### `/dispute` - Adversarial Debate
**Phases**:
1. Builder proposes implementation approach
2. Auditor challenges assumptions and identifies risks
3. Architect evaluates long-term impact
4. Consensus reached or escalated
5. **Auto-generate audit report** → `memory/auditor-reports/YYYY-MM-DD-topic.md`

**Output**: Dispute Resolution Report with decision link (D0XX)

#### `/maintenance` - System Entropy Reduction
**Phases**:
1. **Integrity Audit**: Run check-harness.py for structural validation
2. **Standardization**: Remove industry-standard advice from AGENTS.md
3. **Knowledge Layering**: Move troubleshooting to knowledge-base with quality gate
4. **Usage Tracking**: Update SKILL usage stats, identify extraction candidates
5. **System Logs Management**:
   - Generate maintenance log → `memory/maintenance-logs/YYYY-MM-DD-maintenance-log.md`
   - Archive auditor reports (> 90 days) → `memory/archives/auditor-reports-YYYY-QN.md`
   - Delete old maintenance logs (> 60 days)

**Output**: [ENTROPY_REPORT], [PRUNING_PROPOSAL], [HEAT_MAP], [LEAN_UPDATE]

#### `/update-memory` - Memory Synchronization
**Phases**:
1. Load mandatory context (core-facts, decisions, progress, AGENTS)
2. Analyze current session work
3. Update progress.md, decisions.md
4. Create session summary
5. Intelligent compression (if triggered)

**Output**: Updated memory files, session summary in `memory/session-summaries/`

---

## 🛡️ Critical Guardrails

All work must adhere to these **Hard Prohibitions** (defined in `AGENTS.md`):

1. **Environment Lock**: ALL Python operations MUST use `uv run` (never system Python)
2. **Architect Alert**: Modifications to D0XX decisions or database schemas trigger mandatory review
3. **No Silent Failures**: Every error must be logged and analyzed
4. **Evidence Mandatory**: No task "Complete" without Verification Evidence block (data flow + test output)

---

## 🔄 Maintenance Lifecycle

Prevent instructional drift and maintain professional density:

- **Standardization**: Keep `AGENTS.md` < 300 lines
- **Layering**: Move one-off fixes to `reference/knowledge-base/`
- **Pruning**: Archive dormant SKILLs (unused 60+ days) via `/maintenance`
- **System Logs**: Auto-manage audit trails and maintenance logs

### System Logs Lifecycle

| Directory | Generated By | Retention | Cleanup Strategy |
|-----------|--------------|-----------|------------------|
| `auditor-reports/` | `/dispute` Step 6 | 90 days | Archive to quarterly files |
| `maintenance-logs/` | `/maintenance` Phase 5.1 | 60 days | Delete (keep last 10) |

**See**: `reference/knowledge-base/SYSTEM_LOGS_MANAGEMENT.md` for complete mechanism.
