# PegBot 持仓和关注列表快捷查询功能

## 功能概述

手机端用户现在可以通过飞书 PegBot 卡片命令或文字命令,快速获取持仓列表和动态关注列表,无需多次点击进入管理菜单。

## 实现方式

### 1. 主菜单卡片快捷按钮

在 PegBot 主菜单卡片中新增了两个快捷查看按钮:

- **📈 查看持仓** → 点击后直接显示持仓列表
- **👀 查看关注** → 点击后直接显示关注列表

原有的管理按钮保留:
- **💼 持仓管理** → 进入持仓管理菜单(添加/删除股票)
- **📋 关注管理** → 进入关注列表管理菜单(添加/删除关注)

### 2. 文字命令支持

用户可以直接发送文字命令查询:

| 命令 | 别名 | 功能 |
|------|------|------|
| `持仓` | `自持`、`自持股票` | 查看持仓列表 |
| `关注` | `动态关注` | 查看今日关注列表 |
| `查询 [股票名称]` | `行情 [代码]` | 查询单只股票 |
| `帮助` | `help` | 查看帮助信息 |

## 技术实现

### 修改的文件

1. **backend/services/feishu_card_service.py**
   - 主菜单卡片添加快捷查看按钮
   - 按钮action直接调用 view_holdings 和 view_watchlist

2. **backend/services/feishu_bot.py**
   - 新增命令解析:持仓/自持/关注/动态关注
   - 新增处理方法:_handle_view_holdings() 和 _handle_view_watchlist()
   - 更新帮助信息

3. **backend/services/daily_watchlist_manager.py**
   - 新增同步查询方法:get_watchlist_by_date_sync()

4. **backend/routers/feishu.py**
   - 已有 view_holdings 和 view_watchlist 的处理逻辑

### 数据来源

- **持仓列表**: 从 `backend/data/自持股票.md` 文件读取
- **关注列表**: 从数据库表 `daily_watchlist` 读取今日未归档的股票

### 数据流程

```
用户操作
  ↓
飞书Webhook/长连接
  ↓
feishu.py: process_card_callback_event() 或 process_message_event()
  ↓
判断action类型
  ├─ view_holdings → feishu_bot._handle_view_holdings()
  │   ↓
  │   holdings_manager.read_holdings() → 读取自持股票.md
  │   ↓
  │   格式化输出
  │
  └─ view_watchlist → feishu_bot._handle_view_watchlist()
      ↓
      daily_watchlist_manager.get_watchlist_by_date_sync()
      ↓
      数据库查询今日关注列表
      ↓
      格式化输出
```
