# 股票资料目录说明

本目录存储各股票的详细资料，采用Markdown格式存储。

## 目录结构

```
stock_profiles/
├── README.md           # 本说明文件
├── 600219.md          # 南山铝业
├── 601088.md          # 神华股份
├── {股票代码}.md       # 其他股票资料
└── ...
```

## 文件命名规范

- 文件名：`{股票代码}.md`（例如：`600219.md` - 南山铝业）
- 编码：UTF-8
- 格式：Markdown

## 内容格式

每个股票资料文件应包含以下章节：

```markdown
# [股票名称] ([股票代码])

## 最新年报或季报

**报告类型**: 年报/季报
**报告期**: YYYY-MM-DD
**公告日期**: YYYY-MM-DD

### 财务摘要
- 营业收入: XX亿元
- 净利润: XX亿元
- 基本每股收益: XX元
- 经营现金流: XX亿元

### 经营分析
（管理层讨论与分析内容）

## 公司概况

### 基本信息
- 上市日期: YYYY-MM-DD
- 所属行业: XX
- 主营业务: XX

### 业务亮点
（公司核心竞争优势、业务亮点）

## 投资要点

（分析师观点、投资建议）

## 风险提示

（主要风险因素）

---
*最后更新: YYYY-MM-DD HH:MM:SS*
*数据来源: 手动维护 / 自动抓取*
```

## 数据更新策略

1. **优先级**：优先使用本目录下的Markdown文件
2. **缺失时**：如果文件不存在，从API获取数据
3. **自动更新**：定期从API获取最新数据并更新文件
4. **手动维护**：支持手动编辑补充内容

## API接口

- GET `/api/stock/profile/{stock_code}` - 获取股票资料
- PUT `/api/stock/profile/{stock_code}` - 更新股票资料
- POST `/api/stock/profile/{stock_code}/refresh` - 刷新股票资料
